“¿Quién es dueño del código?” debe responderse antes de escribir una línea. Los defaults cambian por país y tipo de contrato — no confíes en sensaciones.

## Qué deberías poseer

- Código fuente de la aplicación producido para ti.
- Infraestructura como código y configs de deploy del proyecto.
- Credenciales, dominios y recursos cloud para operar el producto.
- Documentación para operar y extender el sistema.

## Fallos comunes

- Código en el GitHub personal del freelancer sin transferencia.
- “Framework propietario” de la agencia que te atrapa.
- Sin cesión de IP escrita en el statement of work.

## Protección práctica

- Crea el repositorio bajo tu org desde el día uno.
- Incluye lenguaje de cesión de IP en el contrato.
- Exige handover de secretos vía tu password manager / cloud — no por chats.

> Si no puedes clonar el repo y redesplegar sin el vendor, todavía no eres dueño del producto.
