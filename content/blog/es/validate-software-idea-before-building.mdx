El software más caro es el que nadie usa. La validación no es teatro opcional — es cómo proteges presupuesto.

## Validación barata antes de ingenieros

- Habla con 10–20 usuarios objetivo sobre el trabajo, no sobre tu lista de features.
- Prueba una landing o waitlist con una oferta clara.
- Corre una versión concierge manual (tú haces el flujo a mano).
- Vende un piloto antes de construir cada pantalla.

## Cómo se ve “suficientemente validado”

- La gente describe el dolor sin que tú empujes la solución.
- Alguien paga, pilotea o compromete tiempo.
- Puedes definir un flujo principal para un MVP.

## Cuándo empezar a construir

Cuando el job-to-be-done esté claro y haya evidencia de demanda, financia una rebanada pequeña entregable — no todo el roadmap.
