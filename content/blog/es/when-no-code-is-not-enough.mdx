Las herramientas no-code te ayudan a avanzar antes de tener capacidad de ingeniería. Eso vale. El error es quedarte en ellas cuando el producto ya es core para ingresos, compliance o confianza del cliente.

## Señales de que superaste el no-code

- Los workarounds crecen más rápido que las features.
- Necesitas IA a medida (RAG, agentes) conectada a sistemas internos.
- Permisos, auditoría o residencia de datos se vuelven requisitos.
- Límites de rendimiento o del vendor frenan el crecimiento.
- Tu equipo pelea más con la herramienta que con el cliente.

## Cómo migrar sin pánico de reescritura

No hace falta rehacerlo todo de golpe. Empieza por el flujo de más riesgo o más ingreso. Mantén no-code en operaciones periféricas mientras construyes el core en un stack real (por ejemplo Next.js + APIs TypeScript + base de datos adecuada).
