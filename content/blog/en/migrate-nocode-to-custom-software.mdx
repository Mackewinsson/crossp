Outgrowing no-code is a success problem — until fragile automations start blocking sales, support, or compliance. The fix is rarely “rebuild everything next quarter.”

## Decide what to migrate first

Rank workflows by:
- revenue impact,
- operational risk,
- frequency of breakage,
- need for custom AI, permissions, or audit trails.

Start with the highest score. Leave low-risk ops on no-code until the core is stable.

## Migration pattern that works

1. Map the current workflow and data.
2. Rebuild the core path on a real stack (for example Next.js + APIs + proper database).
3. Keep secondary flows on no-code during transition.
4. Cut over when staging proves the job works end to end.

## What not to do

- Parallel “perfect” rebuilds that never launch.
- Copying every no-code workaround into custom code.
- Switching vendors mid-migration without ownership of credentials.
